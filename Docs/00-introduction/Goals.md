# Goals

> This document defines what Eulinx is intentionally trying to achieve. Every roadmap item should support one or more of these goals.

# Primary Goal

Create the world's best local-first AI Operating System where users manage objectives while intelligent worker terminals execute the work.

---

# Strategic Goals

## Goal 1 — Build an AI Operating System

Eulinx should feel like an operating system for AI work rather than a chatbot or workflow editor.

Success indicators:
- Dynamic worker hierarchy
- Runtime scheduling
- Real execution
- Workspace isolation

---

## Goal 2 — Make AI Actually Do Work

Workers should:
- Write code
- Run commands
- Execute tests
- Research
- Verify outputs
- Produce artifacts

The emphasis is execution, not conversation.

---

## Goal 3 — Stay Local First

The complete experience should work locally whenever possible.

Cloud services are optional enhancements.

---

## Goal 4 — Transparent Execution

Users should always understand:

- What is happening
- Which worker owns a task
- Current progress
- Generated artifacts
- Runtime events
- Costs

Nothing important should happen invisibly.

---

## Goal 5 — High Quality Through Collaboration

Instead of relying on one large prompt, Eulinx improves quality by:

- Planning
- Decomposition
- Parallel workers
- Refinement loops
- Verification
- Controlled merging

---

## Goal 6 — Safe Automation

Automation should never sacrifice safety.

All destructive operations should support:

- Permission checks
- Human approval
- Verification
- Rollback

---

## Goal 7 — Scalable Architecture

The architecture should comfortably evolve from:

1 worker

to

hundreds of workers

without redesigning the core runtime.

---

## Goal 8 — AI-Friendly Development

Documentation, architecture, and code organization should be understandable by lower-cost coding models.

Every system should have:

- Clear responsibilities
- Predictable structure
- Rich documentation
- Minimal ambiguity

---

## Goal 9 — Extensible Platform

Future additions should require minimal changes to existing systems.

Examples:

- New AI providers
- Plugins
- MCP servers
- Node types
- Runtime services
- Marketplace content

---

## Goal 10 — Long-Term Platform

Eulinx should evolve into a complete platform for:

- Software engineering
- Research
- Automation
- Knowledge management
- AI collaboration
- Business workflows

---

# Success Metrics

A successful Eulinx release should:

✓ Reduce manual prompting

✓ Increase task completion

✓ Improve output quality

✓ Reduce context waste

✓ Keep users in control

✓ Scale without architectural rewrites

---

# AI Notes

When implementing new features, identify which strategic goal(s) they support.

If a feature supports none of these goals, reconsider whether it belongs in Eulinx.

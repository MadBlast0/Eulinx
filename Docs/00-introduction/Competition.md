# Competition

> This document explains how Eulinx relates to existing products. The goal is not to copy competitors, but to understand their strengths, weaknesses, and where Eulinx creates a new category.

# Philosophy

Do not ask:

"How do we beat Product X?"

Ask instead:

"What problem is still unsolved?"

Eulinx should solve that problem.

---

# AI Chat Applications

Examples:
- ChatGPT
- Claude
- Gemini

Strengths
- Excellent reasoning
- Easy to use
- General purpose

Weaknesses
- Single conversation
- Manual context management
- Little execution orchestration

Eulinx Difference

User gives a goal.

Eulinx builds and coordinates an execution hierarchy.

---

# AI Coding Assistants

Examples

- Cursor
- Claude Code
- OpenCode
- Codex CLI

Strengths

- Great coding assistance
- Terminal integration
- Project awareness

Weaknesses

- Mostly one active execution context
- User coordinates parallel work

Eulinx Difference

Multiple worker terminals collaborate automatically.

---

# Workflow Builders

Examples

- n8n
- Flowise
- Langflow
- Make

Strengths

- Visual automation
- Integrations
- Triggers

Weaknesses

- AI is usually just another node.
- Complex graphs become difficult to manage.

Eulinx Difference

The runtime graph visualizes living AI execution rather than static automation.

---

# Multi-Agent Frameworks

Examples

- AutoGen
- CrewAI
- LangGraph

Strengths

- Powerful orchestration
- Flexible

Weaknesses

- Developer-oriented
- Code-first
- Limited visual observability

Eulinx Difference

Desktop-first, visual, interactive, local-first.

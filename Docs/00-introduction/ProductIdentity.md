# Product Identity

> Product Identity defines what Eulinx fundamentally is, how it should be perceived, and what it should never become.

# Identity Statement

Eulinx is a **local-first AI Operating System** that coordinates intelligent worker terminals to execute real work safely, visually, and transparently.

It is **not** a chatbot, a prompt manager, or a simple workflow builder.

---

# Positioning

Instead of competing directly with AI chat applications, Eulinx creates a new category:

AI Operating System (AIOS)

The user manages goals.

The runtime manages execution.

Workers perform work.

Runtime services guarantee correctness.

---

# The User's Mental Model

The user should feel like:

- a project manager
- a technical lead
- a studio director

The user gives objectives rather than micromanaging every prompt.

---

# Product Personality

Eulinx should feel:

- Calm
- Professional
- Transparent
- Predictable
- Powerful
- Trustworthy

Avoid feeling:

- Gimmicky
- Noisy
- Magical
- Overly automated
- Hidden

---

# Core Experience

The expected experience is:

User
↓
Choose Workspace
↓
Describe Goal
↓
Runtime Creates Plan
↓
Workers Spawn Dynamically
↓
Artifacts Flow
↓
Verification
↓
Merge
↓
Completed Result

The user can intervene at any stage.

---

# Design Identity

Visual style should communicate:

- Engineering
- Reliability
- Organization
- Live execution
- Real-time observability

Motion exists to explain execution, never simply for decoration.

---

# Competitive Identity

Eulinx is different because:

- Workers execute inside real CLI terminals.
- The runtime graph grows dynamically.
- Projects are isolated by workspace.
- AI collaboration is visual.
- Runtime services enforce safety.
- Artifact-based communication minimizes context waste.
- Local-first architecture avoids vendor lock-in.

---

# Long-Term Identity

Eulinx should eventually become the primary environment where users:

- build software
- automate businesses
- research topics
- coordinate AI workers
- create reusable workflows
- manage knowledge

The application should evolve into an extensible operating environment rather than a collection of disconnected features.

---

# Identity Rules

Always preserve these ideas:

1. Local First
2. Runtime Before AI
3. Transparency Over Mystery
4. Workers Over Chats
5. Artifacts Over Conversations
6. Safety Over Convenience
7. Extensibility Over Hardcoding

---

# AI Notes

When implementing features:

- Ask "Does this strengthen the identity of an AI Operating System?"
- Avoid features that turn Eulinx into a generic chat application.
- Every feature should reinforce execution, orchestration, and observability.

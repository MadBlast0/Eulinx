# NonGoals

> A great product is defined as much by what it refuses to become as by what it includes.

## Purpose

This document defines the boundaries of Eulinx. Any proposed feature that falls into these categories should be questioned before implementation.

---

# Non-Goal 1 — Another AI Chat App

Eulinx is not intended to compete as a traditional chat interface.

Conversation supports work, but conversation is never the final product.

---

# Non-Goal 2 — Replace Foundation Models

Eulinx does not train, host, or compete with LLM providers.

It orchestrates existing models through local or remote providers.

---

# Non-Goal 3 — Become a Cloud-Only Platform

Cloud synchronization, hosted models, and collaboration are optional.

Local-first remains the default.

---

# Non-Goal 4 — No Hidden Automation

The runtime should never silently modify projects without visibility.

Execution should always be observable.

---

# Non-Goal 5 — Permanent AI Personalities

Workers are temporary execution units.

Behavior comes from assigned tasks rather than fixed personalities.

---

# Non-Goal 6 — Unlimited Permissions

Workers should not automatically receive unrestricted access.

Permissions must remain explicit and configurable.

---

# Non-Goal 7 — Monolithic Context

Do not solve complexity by continuously expanding context windows.

Prefer decomposition, artifacts, summaries, and worker hierarchies.

---

# Non-Goal 8 — Replace Human Judgment

Humans remain responsible for important architectural and business decisions.

Eulinx assists execution rather than replacing ownership.

---

# Non-Goal 9 — Vendor Lock-In

Users should be free to:

- choose providers
- export projects
- import templates
- replace models
- extend functionality

---

# Non-Goal 10 — Feature Creep

Features that don't strengthen Eulinx as an AI Operating System should not be added simply because competitors have them.

---

# Scope Boundaries

Out of scope for the core product:

- Social media platform
- General-purpose website builder
- Foundation model training
- Closed ecosystem
- Mandatory subscriptions
- Proprietary workflow format

---

# AI Notes

When proposing new functionality ask:

1. Does this reinforce Eulinx's identity?
2. Does it conflict with any non-goal?
3. Can it remain optional?
4. Does it preserve local-first principles?

If the answer is no, reconsider the feature.

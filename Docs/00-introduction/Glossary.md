# Glossary

> Quick A–Z reference for important terms, technologies, acronyms and concepts used throughout the Eulinx documentation.

## A

### AI
Artificial Intelligence used for reasoning and task execution.

### Artifact
A structured output produced by a worker such as a patch, report, markdown document, JSON, test result, or image.

### API
Application Programming Interface.

## C

### Context
Information provided to a worker before execution.

### CLI
Command Line Interface.

## E

### Event Bus
Runtime messaging system connecting runtime services, workers, plugins and UI.

## G

### Git
Version control system.

## K

### Eulinx
A local-first AI Operating System built around worker terminals, orchestration and runtime services.

### Knowledge Base
Indexed project knowledge available to workers.

## L

### Local First
Architecture where the user's machine is the primary execution environment.

### LLM
Large Language Model.

## M

### MCP
Model Context Protocol.

### Memory
Persistent or temporary information available during execution.

### Merge
Applying verified artifacts to the workspace.

## N

### Node
Visual representation of a runtime object.

## O

### Orchestrator
Planning component responsible for decomposing work and coordinating workers.

## P

### Phase
A major project milestone managed by a Phase Orchestrator.

### Plugin
An extension adding new capabilities.

### Provider
AI service supplying language models.

### PTY
Pseudo Terminal used to run real terminal sessions.

## R

### Runtime
Execution engine coordinating workers and runtime services.

### Runtime Service
Deterministic software responsible for scheduling, permissions, locking, events and merging.

## S

### Scheduler
Runtime component assigning work to workers.

### Session
One execution period inside a workspace.

## T

### Task
A unit of work.

### Tool
A capability available to workers.

## V

### Verification
Validation of artifacts before merging.

### Vector Database
Database optimized for semantic embeddings.

## W

### Worker
Temporary AI-powered terminal executing a specific objective.

### Workspace
An isolated project environment containing files, workers, memories, history and runtime state.

# Purpose

LongTermMemory stores durable information that should survive beyond one Session.

It should contain stable facts, not raw work history.

# Durable Facts

Good LongTermMemory:

- project architecture rule
- user preference
- recurring command
- important decision
- known failure mode

Bad LongTermMemory:

- random terminal output
- speculative plan
- one-time error
- sensitive secret

# Promotion

LongTermMemory should be created through promotion from lower-scope memory or explicit user entry.
